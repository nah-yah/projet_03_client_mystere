## 3. Matrice de rotation des scénarios

### Règles de rotation

Cinq règles s'appliquent sans dérogation à toutes les agences et tous les territoires.

1. Chacun des cinq scénarios couvre exactement une visite par agence par vague.
2. Aucun enquêteur ne visite la même agence deux fois dans la même vague.
3. Les visites se distribuent sur les trois créneaux: matin 8h à 12h, midi 12h à 14h, après-midi 14h à 17h.
4. Deux visites à la même agence ne tombent pas le même jour calendaire.
5. Chaque enquêteur opère dans son territoire de résidence.

## 4. Décisions affectant la stratification

### DEC-01: Charge enquêteur Guadeloupe

Le pool GP comprend 5 enquêteurs pour 8 agences, soit 40 visites à distribuer. Chaque enquêteur GP effectue 7 à 8 visites, contre 5 en Martinique et exactement 5 en Guyane française. Cette charge supérieure est non bloquante mais à surveiller : un abandon en cours de vague nécessite un remplacement immédiat dans le pool GP. Aucun enquêteur GP ne dépasse 8 visites dans la Vague 1.

### DEC-02: G4 singleton

BCA-MQ-11 (Le Vauclin, 5 ans) est la seule agence du groupe G4. Cette situation interdit toute comparaison statistique impliquant G4 comme groupe à part entière. Dans les analyses, BCA-MQ-11 apparaît dans les tableaux descriptifs mais en est exclue dès qu'une comparaison inter-groupes est effectuée. La Vague 2 ne devrait pas modifier ce statut, BCA-MQ-11 ayant 6 ans à ce moment.

### DEC-13: Reclassification BCA-GP-06

BCA-GP-06 (Saint-François, Guadeloupe) a été ouverte en 2011, ce qui donne 14 ans d'ancienneté en 2025. Le seuil inférieur du groupe G2 est fixé à 15 ans. BCA-GP-06 tombe donc en G3 (6 à 14 ans). La classification dans les fichiers de données, les matrices de rotation et tous les scripts reflète cette décision. Toute correction vers G2 fausserait les tests d'association ancienneté/performance.

## 5. Dictionnaire des variables

Variables du fichier `02_donnees/brutes/metadata_agences.csv`.

| Variable | Type R | Description | Valeurs |
|---|---|---|---|
| `id_agence` | chr | Identifiant unique | BCA-MQ-01 à BCA-GF-05 |
| `nom_agence` | chr | Nom complet de l'agence | Texte libre |
| `territoire` | chr | Territoire administratif | Martinique, Guadeloupe, Guyane francaise |
| `code_territoire` | chr | Code à 2 lettres | MQ, GP, GF |
| `ville` | chr | Ville d'implantation | Texte libre |
| `tier` | int | Tier de l'agence | 1, 2, 3 |
| `annee_ouverture` | int | Année d'ouverture | 1989 à 2020 |
| `anciennete_ans` | int | 2025 moins annee_ouverture | 5 à 36 |
| `groupe_anciennete` | chr | Groupe d'ancienneté | G1, G2, G3, G4 |
| `label_anciennete` | chr | Label du groupe | Historique, Etablie, Recente, Tres recente |
| `agence_fondatrice` | lgl | Première agence du territoire | TRUE, FALSE |
| `visites_vague1` | int | Visites planifiées (Vague 1) | 5 (constant) |
| `latitude` | dbl | Latitude GPS du centroïde (WGS84) | Décimal |
| `longitude` | dbl | Longitude GPS du centroïde (WGS84) | Décimal |
| `rayon_geofence_m` | int | Rayon de la géofence de validation GPS | 100 m (constant) |
| `nb_agents_estime` | int | Agents en poste (estimation) | 2 à 15 |
| `zone_type` | chr | Type de zone d'implantation | Urbaine dense, Urbaine, Peri-urbaine, Rurale |

Les valeurs textuelles dans le CSV utilisent l'encodage ASCII sans accents (DEC-07) pour garantir la portabilité entre systèmes. Les labels avec accents sont appliqués dans les scripts d'analyse lors de la création des variables d'affichage.
